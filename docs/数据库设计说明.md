## 2. 设计原则

### 2.1 核心原则

1. **数据完整性**：使用外键约束、CHECK约束、NOT NULL约束确保数据质量
2. **安全性**：启用Row Level Security (RLS)，用户只能访问自己的数据
3. **可扩展性**：预留扩展字段（JSONB），支持未来功能迭代
4. **性能优化**：合理的索引策略，支持全文检索
5. **审计追踪**：所有表包含created_at和updated_at时间戳

### 2.2 命名规范

| 对象类型 | 命名规范 | 示例 |
|----------|----------|------|
| 表名 | 小写，复数形式，下划线分隔 | `users`, `dream_entries` |
| 字段名 | 小写，下划线分隔 | `user_id`, `created_at` |
| 索引名 | `idx_表名_字段名` | `idx_dreams_user_id` |
| 约束名 | `fk_表名_引用表` | `fk_dreams_users` |
| 函数名 | 小写，动词开头 | `increment_quota` |

## 4. 数据表设计

### 4.1 用户相关表

#### users（用户主表）

**表说明**：存储用户基础信息，支持注册用户和匿名用户

| 字段名 | 数据类型 | 约束 | 默认值 | 中文说明 |
|--------|----------|------|--------|----------|
| id | UUID | PRIMARY KEY | gen_random_uuid() | 用户唯一标识 |
| phone | VARCHAR(20) | UNIQUE, 可空 | NULL | 手机号，注册用户必填 |
| password_hash | VARCHAR(255) | 可空 | NULL | 密码哈希值 |
| nickname | VARCHAR(100) | 可空 | NULL | 用户昵称 |
| avatar_url | TEXT | 可空 | NULL | 头像URL地址 |
| gender | VARCHAR(10) | CHECK | NULL | 性别：male/female/other |
| age | INTEGER | CHECK(0-150) | NULL | 年龄 |
| is_anonymous | BOOLEAN | NOT NULL | FALSE | 是否为匿名用户 |
| device_id | VARCHAR(255) | 可空 | NULL | 匿名用户设备标识 |
| last_login_at | TIMESTAMP | 可空 | NULL | 最后登录时间 |
| created_at | TIMESTAMP | NOT NULL | NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL | NOW() | 更新时间 |

## 5. Row Level Security (RLS) 策略

所有业务表均启用RLS，确保数据隔离：

| 表名 | RLS策略说明 |
|------|------------|
| users | 用户只能访问自己的数据 (id = auth.uid()) |
| user_profiles | 用户只能访问自己的档案 (user_id = auth.uid()) |
| subscriptions | 用户只能访问自己的订阅 (user_id = auth.uid()) |
| dream_entries | 用户只能访问自己的梦境 (user_id = auth.uid()) |
| audio_profiles | 用户只能访问自己的音频 (user_id = auth.uid()) |
| dream_tags | 用户只能访问自己的标签 (user_id = auth.uid()) |
| media_assets | 用户只能访问自己的媒资 (user_id = auth.uid()) |
| long_video_scripts | 用户只能访问自己的剧本 (user_id = auth.uid()) |
| ai_generation_tasks | 用户只能访问自己的任务 (user_id = auth.uid()) |
| interpretations | 用户只能访问自己的解读 (user_id = auth.uid()) |
| knowledge_items | 所有用户可查看有效的知识条目 (status = 'active') |
