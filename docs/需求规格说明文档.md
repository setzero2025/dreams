# 需求规格说明文档

## 1. 文档信息
- **产品名称**：梦境空间（hasDream）移动应用
- **版本**：v0.1 草案（2026-04-02）
- **编写目的**：为交互设计、研发、测试与评审提供统一的功能与体验基线，指导 iOS / Android 双平台实现。
- **面向角色**：产品经理、交互/视觉、客户端开发、后端/算法、QA、运维。

## 2. 产品概述
### 2.1 背景
中国用户对梦境记录、解析与二次创作的需求逐渐增多，当前缺乏一体化记录-生成-分享-解析闭环体验。本应用以“记梦、生梦、圆梦、享梦、解梦”为主轴，并提供内容管理、订阅及友商联动能力。

### 2.2 目标
1. 提供文字/语音两种低门槛的梦境采集方式。
2. 借助多款大模型能力将梦境转化为图片、短视频、长剧情视频。
3. 提供梦境分享、解读与自我心理测评工具。
4. 建立多媒体资产管理、知识库管理及订阅付费体系，保证商业化能力与数据安全。

### 2.3 设计原则
- **交互简洁**：核心路径 3 步内完成（如记梦→生梦（图片、视频、长剧情视频）→分享）。
- **移动原生习惯**：底部 4 栏主导航 ，遵循 iOS HIG 与 Android Material。
- **云本地协同**：登录后所有资产云端保存；未登录仅本地缓存，卸载后清空。
- **智能提示**：所有 AI 生成流程需展示进度、模型来源及预计耗时。

## 3. 用户与场景
| 角色 | 特征 | 主要诉求 | 关键功能 |
| --- | --- | --- | --- |
| 访客（未登录） | 首次体验、无账号 | 快速体验记梦与生成效果 | 文字/语音记梦、1 张图、1 个视频预览、仅本地存储 |
| 注册用户（未订阅） | 有账号未付费 | 长期记录，有限生成名额 | 无限文字/语音记录、梦图最多 5 张、梦视频 2 个、长视频 1 个 |
| 订阅用户 | 按月/年付费 | 大量生图/视频、专业解梦 | 无限生成、测评、跨设备同步、AI心理小伴互通 |

## 4. 系统概述
### 4.1 客户端结构
- **底部导航**：首页 / 记梦 / 资产 / 我的。
- **离线策略**：记录时离线草稿缓存，恢复网络后自动同步。

### 4.2 服务与数据流
1. 语音录入 → 讯飞实时语音转写 → 文本持久化。
2. 梦境描述 → 豆包 seedream 4.5 → 图片 → 媒资服务 → 本地/云展示。
3. 梦境描述 → 阿里百炼 Wan2.6-T2V → 5s 视频片段。
4. 场景剧本 → kimi K2.5 → 场景脚本 → 豆包 seedream 4.5 → 关键帧图 → 阿里百炼 Wan2.6-T2V → 5s 视频片段 → 第三方视频合成服务（按 4 段拼接 20s）。
5. 梦境文本 → 内部知识库 + LLM → 解梦与测评报告。

### 4.3 数据实体
- DreamEntry（标题、文字、情绪标签、时间、语音引用）
- AudioProfile（音色、音色特征）
- MediaAsset（图片/视频/长视频/音色、关联 DreamEntry、关联LongVideoScript、生成来源）
- LongVideoScript（剧本、关联 DreamEntry、剧本内容）
- Interpretation（解梦/测评类型、参考知识、建议）
- KnowledgeItem（主题、来源、可用性）
- Subscription (订阅, start/end, 配额)
- User（用户，昵称、手机号、密码、性别、年龄）
- UserProfile（用户信息，本次订阅或未订阅期间已使用各类额度）

## 5. 功能需求
### 5.1 首页模块
1. **整体布局**
	- 头部显示欢迎语：梦境空间，记录你的梦境，探索潜意识
	- 欢迎语下面是三个tab页：今天、本周、全部，默认显示今天，与下方的梦境列表联动
	- tab页下面是带搜索框的梦境列表，可根据梦境名称搜索
2. **梦境列表**
	- 记梦首页的下面展示今天、本周、全部的梦境列表，按梦境保存时间倒序排序。
	- 点击列表中的某一个梦境，跳转到梦境详情页面。
	- 右滑某一梦境，显示“删除”功能，点击并确认后可删除该梦境及梦境相关的所有图片、视频、解读结果。
3. **梦境详情**
	- 梦境详情页面展示梦境提交的梦境标题、内容、情绪选择、梦境标签、录音，录音可回放。
	- 详情下方，直接展示“梦生图片”“梦生视频”“梦境剧情”“梦境解读”四个快捷功能。
	- “梦生图片”“梦生视频”“梦境剧情”“梦境解读”四个快捷功能的下方展示该梦境相关的梦境作品。
	- 梦境详情页面右上角有“分享”功能，可以分享朋友圈或小红书。
4. **梦生图片**
   - 根据当前梦境生成梦境图片；可指定风格（国潮、赛博、油画、动漫、科幻等）。
   - 提交后调用豆包 seedream 4.5，展示排队/生成进度条。
   - 生成结果默认 9:16，可选择保存、设为封面、进入分享流程。
   - 限制：未订阅用户仅 5 个名额，其余提示“订阅后可继续创作”。
5. **梦生视频**
   - 默认长度 5s，可选择纵向/横向比例。
   - 调用阿里百炼 Wan2.6-T2V 接口，失败提供错误码与重试。
   - 限制：未订阅用户仅 2 个名额，其余提示“订阅后可继续创作”。
   - 生成的视频进入视频管理，支持下载、分享、删除。
6. **梦境剧情**
   - 步骤：根据当前梦境内容描述 → kimi K2.5 生成 4 场景剧本 → 每场景生成关键帧图 → 每图生成 5s 视频 → 视频拼接 20s。
   - UI：流程分步展示（剧本、关键帧、视频、合成）并可回看中间产物。
   - 产出：20s 视频 + 剧本文本，保存至视频管理。
   - 限制：未订阅用户仅 1 个名额，其余提示“订阅后可继续创作”。
7. **梦境解读**
   - 针对当前梦境内容，系统调用大模型（自研+知识库）输出含符号解读、情绪分析及建议。
   - 展示引用的知识库条目（标题+来源）。
8. **梦境作品**   
	- 如果当前梦境生成了梦境图片、视频、剧情视频和梦境解读文字，则在记梦首页最底部展示“梦境作品”模块。
	- 梦境作品模块按照创建时间倒序展示作品列表，每个作品标明类型。
	- 作品如果是图片则展示图片缩略图，点击可查看详情；
	- 作品如果是视频则展示视频封面图，点击可播放；
	- 作品如果是剧情长视频则展示视频封面图，点击可播放；
	- 作品如果是梦境解读结果，则展示代表文字描述的图片，点击查看解读全内容。
8. **梦境分享**
	- 可以选择“梦境作品”中的某些作品分享朋友圈或小红书。
   8.1 ***朋友圈分享***
   - 可选择文本 + 图片/视频组合。
   - 受限于平台规则，本期仅提供交互模拟：点击“分享到朋友圈”→弹出模拟卡片→提示“分享成功（模拟）”。
	8.2 ***小红书分享***
   - 同朋友圈逻辑，额外提示建议的文案标签。
   - 未来版本可接入系统分享组件，需保留入口与成功 toast。

### 5.3 资产模块
1. **照片管理**：查看梦境生成的图片，允许单条删除与下载。
2. **音频管理**：查看按时间排序的语音、音色记录，允许单条删除与下载。
3. **视频管理**：列出所有生成的视频/长视频，支持收藏、删除、导出本地。
4. **知识库管理**：后台角色可新增/下线知识项，前台展示只读来源信息以提升透明度。

### 5.4 我的模块
1. **个人信息**：
	- "我的"首页顶部个人头像和基本信息，分不同情况展示：
		- **未登录用户**：展示默认头像图像，昵称显示"匿名账户"，手机号位置显示"登录"快捷按钮，点击跳转到登录页面
		- **登录未订阅用户**：展示用户自定义头像（如无自定义头像则展示默认头像），显示用户自己的昵称和手机号（中间4位替换为*展示），同时显示"订阅会员"快捷按钮，点击跳转到订阅页面
		- **登录已订阅用户**：展示用户自定义头像（如无自定义头像则展示默认头像），显示用户自己的昵称和手机号（中间4位替换为*展示），同时展示会员有效期至日期（如：会员有效期至 2025年12月31日）
	- 头像和昵称下面是简单的统计信息：累计梦境、梦境作品、连续天数，连续天数根据梦境日期计算
	- 点击头像支持用户选择相册中的图片更换头像（仅已登录用户可更换，未登录用户点击头像跳转到登录页面）
	- 点击昵称，展示用户个人信息，如昵称、手机号、性别、年龄，其中昵称、性别、年龄可修改（仅已登录用户可编辑）
	- 退出登录按钮仅在已登录状态下显示
2. **我的收藏**
	- 在“个人信息”模块下方第一个菜单是“我的收藏”，展示收藏作品列表。
3. **心理测评**：
	- 注册用户才有该模块功能
	- 注册未订阅用户默认只能分析最近7天的梦境集合，整体做心理测评分析
	- 订阅用户支持用户选择最近7天、15天、30天的梦境集合，整体做心理测评分析
	- 分析结果给出测评结果、改善建议等
4. **主题设置**：
	- 支持三种模式“跟随系统”“浅色模式”“深色模式”，默认选择“跟随系统”
	- 客户选择自己的主题后，系统记录客户主题，自动渲染客户所选主题效果。

6. **退出登录**：退出该系统
7. **登录**
	- 登录页面可以根据手机号和密码进行登录，支持注册功能
	- 注册时需要填写手机号、密码、确认密码，手机号作为系统用户唯一标识，密码规则要求6位以上。

### 5.7 用户认证
1. **未登录体验**
   - 可记录梦境、生成 1 张图+1 个短视频，长视频不可用。
   - 数据仅存本地 SQLite + 媒体沙箱，卸载清空。
   - 后端获取手机设备唯一值作为该用户的唯一标识来注册一个匿名用户，以防客户卸载后重新安装来刷新额度
2. **注册**
   - 手机号 + 密码（≥8 位字母数字组合），暂时不用验证码二次校验。
3. **登录**
   - 手机号+密码，支持多端登录，若生成额度已满提示订阅。
   - 登录后全部资产云端持久化，可跨设备同步。

### 5.8 订阅模块
1. **订阅方式**：月订 19.9 元 / 年订 99.9 元。
2. **支付**：接支付宝 SDK（本期只保留弹窗与“支付暂未实现”提示，流程埋点齐全）。
3. **权益**：移除生成数量上限、解锁长视频无限次、开启测评时间范围选择与 AI心理小伴互通。

### 5.9 相关 App 联动
- **AI心理小伴**：在“我的-关联服务”中绑定，点击跳转互通（DeepLink 或 Scheme），共享用户 ID 以便后续跨产品成长体系。

## 6. 交互与流程要点
1. **跨平台一致性**：共用信息架构与文案，iOS 使用原生导航返回，Android 使用 Material FAB 与沉浸式状态栏。
2. **生成状态反馈**：所有调用 AI 的流程需展示：模型名称、预计时长、进度/排队状态、失败原因。
3. **权限与引导**：拍照/录音首次调用须解释用途；订阅引导在用户触达上限时触发底部弹层。
4. **分享模拟**：朋友圈/小红书交互需保留完整 UI（文案输入、封面预览、模拟 toast）。

## 7. 数据与权限
### 7.1 数据表 & 存储
| 实体 | 主要字段 | 存储 | 备注 |
| --- | --- | --- | --- |
| DreamEntry | id, userId, title, text, emotion[], tags[], audioId, createdAt | 云端 PostgreSQL + 本地 cache | text 支持全文检索 |
| AudioProfile | id, dreamId, pcmUrl, toneVector | 对象存储 + 元数据表 | toneVector 用于音色记录 |
| MediaAsset | id, dreamId, type(image/video), longVideoScriptId, url, modelSource, ratio | OSS | 关联 DreamEntry |
| LongVideoScript | id, scriptJson, scenes[], status | Mongo/JSON | 存储多阶段状态 |
| Interpretation | id, dreamId, type(interp/eval), content, references[] | PostgreSQL | references 指向 KnowledgeItem |
| KnowledgeItem | id, title, source, status | PostgreSQL | 后台 CRUD |
| Subscription | userId, plan, startAt, endAt, status | PostgreSQL | 影响额度 |
| User | id, name, phone, password, sex, age | PostgreSQL | 
| UserProfile | id,userId, imageQuota,videoQuota,longVideoQuota,interpretationQuota | PostgreSQL | 影响额度 |

## 9. 非功能需求
- **性能**：AI 生成接口超时 60s，超时提示并支持后台生成通知；普通页面 <1.5s 首次渲染。
- **安全**：所有请求 https，登录态使用 JWT + 双端加密存储；录音需显式同意。
- **隐私合规**：提供《隐私政策》《用户协议》，支持用户导出与删除梦境数据。
- **多语言**：面向中国用户，界面中文，后续需预留 i18n 架构。
- **可访问性**：重要按钮提供文字+图标，语音进度附带字幕。
- **兼容性**：iOS 15+、Android 10+。
- **存储**：图片、音频、视频等附件可以保存在服务器本地或OSS服务器，通过后端开关控制使用哪种模式

## 10. 运维与监控
- 埋点：记录各生成步骤耗时、失败原因、转化漏斗（记梦→生成→分享）。
- 日志：语音/视频生成日志保留 30 天，含 requestId 与模型版本。
