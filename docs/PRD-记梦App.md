# 记梦App 产品需求文档 (PRD)

**版本**: v1.0  
**日期**: 2026-03-27  
**技术栈**: React Native + Expo + TypeScript  
**平台**: iOS / Android

---

## 1. 执行摘要

### 1.1 问题陈述
现代人睡眠质量问题日益严重，梦境作为潜意识的表达往往被忽视。用户缺乏便捷的工具记录、分析和分享梦境，导致宝贵的自我认知机会流失。现有笔记类App无法满足梦境特有的多媒体记录（语音、图像、视频）和AI分析需求。

### 1.2 解决方案
打造一款专注于梦境记录的移动应用，支持语音/文字多模态记录，利用AI技术生成梦境图像和视频，提供解梦分析和趋势洞察，帮助用户探索潜意识世界。

### 1.3 成功指标
| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 月活跃用户(MAU) | 10万+ | 数据分析平台 |
| 用户留存率(D7) | ≥35% | 用户行为追踪 |
| 付费转化率 | ≥5% | 订阅数据分析 |
| App Store评分 | ≥4.5星 | 应用商店数据 |
| 语音转文字准确率 | ≥95% | 人工抽样评估 |
| AI生成内容满意度 | ≥80% | 用户反馈问卷 |

## 2. 用户画像与体验

### 2.1 用户画像

#### 画像A：自我探索者（主要用户）
- **年龄**: 22-35岁
- **特征**: 对心理学、灵性成长感兴趣
- **痛点**: 想记录梦境但嫌麻烦，希望获得深度分析
- **使用场景**: 每天早晨记录梦境，周末回顾分析

#### 画像B：创意工作者
- **年龄**: 25-40岁
- **特征**: 设计师、作家、艺术家
- **痛点**: 梦境是创作灵感来源，需要可视化呈现
- **使用场景**: 记录有趣梦境并生成视觉素材

#### 画像C：睡眠质量关注者
- **年龄**: 28-45岁
- **特征**: 有睡眠困扰，关注身心健康
- **痛点**: 想通过梦境了解睡眠质量和心理状态
- **使用场景**: 长期记录并分析梦境趋势

### 2.2 用户故事

#### 记梦模块

**US-001: 语音录入梦境**
> 作为刚睡醒的用户，我想要通过语音快速记录梦境，这样可以在记忆消失前捕捉细节。

**验收标准**:
- 点击说梦按钮后，语音录制在≤500ms内启动
- 支持最长10分钟连续录音
- 录音过程中显示实时波形动画
- 支持暂停/继续功能
- 录音文件保存为AAC格式，码率≥128kbps

**US-002: 语音转文字**
> 作为不方便听语音的用户，我想要将语音自动转为文字，这样可以快速浏览梦境内容。

**验收标准**:
- 语音转文字延迟≤3秒（网络正常）
- 支持中文普通话识别，准确率≥95%
- 支持离线识别（基础功能）
- 转换后文字可编辑
- 保留原始语音文件

**US-003: 音色记录**
> 作为想要个性化体验的用户，我想要记录自己的声音特征，这样可以用于后续AI语音合成。

**验收标准**:
- 录制≥30秒语音样本
- 提取音色特征并本地加密存储
- 显示音色采集进度和质量评分
- 支持重新采集

**US-004: 文字记梦**
> 作为习惯文字记录的用户，我想要用文字记录梦境，这样可以精确描述复杂场景。

**验收标准**:
- 支持富文本编辑（加粗、斜体、列表）
- 自动保存草稿，间隔≤10秒
- 支持插入时间戳标记
- 支持标签分类（噩梦/美梦/ lucid dream等）
- 支持情绪评分（1-5星）

**US-005: 语音记梦（存储）**
> 作为喜欢听自己声音的用户，我想要以语音形式存储梦境，这样可以保留原始情感。

**验收标准**:
- 语音文件本地存储，可选云备份
- 支持播放速度调节（0.5x-2x）
- 支持添加文字摘要
- 语音文件支持导出分享

#### 生梦模块

**US-006: 梦生图**

**验收标准**:
- 生成时间≤30秒（标准画质）
- 支持4种画风选择（写实/油画/水彩/赛博朋克）
- 图像分辨率≥1024x1024
- 支持图像编辑（裁剪、滤镜）
- 生成历史可查看和重新生成

**US-007: 梦生视频（5秒）**

**US-009: 梦生长视频**

#### 用户认证模块

**用户状态与权限矩阵**

| 用户状态 | 梦境记录 | 图片生成 | 视频生成 | 长视频 | 数据存储 |
|----------|----------|----------|----------|--------|----------|

| 已订阅 | 不限次数 | 无限 | 无限 | 无限 | 云端永久保存 |

**US-020: 未登录体验模式**
> 作为新用户，我想要在未登录时免费体验核心功能，这样可以了解App价值后再决定是否注册。

**验收标准**:
- 未登录用户可直接使用记梦功能，数据存储在本地
- 未登录用户可免费生成梦境文字记录
- 未登录用户可免费生成1张梦境图片
- 未登录用户无法使用视频生成功能
- **设备绑定机制**：首次启动时获取设备唯一标识（Device ID），并上报服务端记录
- **额度校验**：每次AI生成前，服务端校验设备ID是否已使用过免费额度
- **防刷策略**：同一设备卸载重装后，服务端根据设备ID识别已使用过免费额度，不再发放新额度
- 登录时提示可将本地数据同步到云端

**设备标识技术方案**:
```
设备ID生成: 使用 react-native-device-info 获取
- Android: 使用 ANDROID_ID 或组合标识
- iOS: 使用 IDFV (Identifier for Vendor)
- 备用方案: 本地生成 UUID 并存储在 Keychain/Keystore

服务端存储:
- 表名: device_quota
- 字段: device_id, image_used, created_at, last_used_at
- 索引: device_id (唯一索引)
```

## 3. AI系统需求

### 3.1 大模型服务架构

采用混合模型策略，根据场景选择最优服务：

| 功能场景 | 推荐模型 | 备选方案 | 选择理由 |
|----------|----------|----------|----------|
| 语音转文字 | 讯飞/阿里云ASR | 腾讯云ASR | 中文识别准确率高 |
| 解梦分析 | Claude/GPT-4 | 文心一言 | 心理学理解能力强 |
| 图像生成 | DALL-E 3/SDXL | 通义万相 | 艺术风格多样 |
| 视频生成 | Runway/Pika | 即梦 | 生成质量高 |
| 剧本生成 | Claude/GPT-4 | 通义千问 | 创意写作能力强 |
| 对话陪伴 | Claude | 文心一言 | 共情能力好 |

### 3.2 AI功能技术规格

#### 语音转文字
```
输入: AAC音频文件 (≤10分钟)
输出: 纯文本 + 时间戳
延迟: ≤3秒 (网络正常)
准确率: ≥95% (普通话)
支持: 离线模式（基础识别）
```

#### 图像生成
```
输入: 梦境文字描述 (≤500字)
输出: PNG图像 1024x1024
生成时间: ≤30秒
风格选项: 写实/油画/水彩/赛博朋克
每日限额: 免费3张，订阅无限
```

#### 视频生成
```
输入: 梦境文字 + 可选照片 + 背景音乐
输出: MP4视频 (5s/10s/60s)
分辨率: 720p/1080p
生成时间: 60s-300s
帧率: 24fps
```

#### 解梦分析
```
输入: 梦境文字 (≥50字)
输出: JSON格式分析报告
{
  "symbols": [...],
  "emotions": {...},
  "interpretation": "...",
  "suggestions": [...]
}
响应时间: ≤10秒
```

### 3.3 评估策略

#### 语音转文字评估
- **测试集**: 100条不同口音、语速的录音
- **通过标准**: 字准确率≥95%，句准确率≥85%
- **人工复核**: 每周抽样10%进行人工校验

#### 图像生成评估
- **测试集**: 50个不同主题的梦境描述
- **通过标准**: 人工评分≥4分（5分制）
- **A/B测试**: 对比不同模型的用户满意度

#### 解梦分析评估
- **测试集**: 30个经典梦境案例
- **通过标准**: 与心理学理论一致性≥80%
- **用户反馈**: 收集用户对解读准确性的评分

interface UserMonthlyQuota {
  user_id: string;
  year_month: string;         // 格式: YYYY-MM
  image_used: number;         // 已用图片额度
  image_limit: number;        // 图片额度上限(免费版5张)

### 4.5 安全与隐私

#### 数据安全
- 所有API通信使用HTTPS/TLS 1.3
- 敏感数据（token、个人信息）本地加密存储
- 语音文件使用AES-256加密
- 定期安全审计和漏洞扫描

#### 隐私合规
- 用户数据最小化收集原则
- 明确的隐私政策和用户协议
- 支持数据导出和删除（GDPR合规）
- AI训练不使用用户个人数据

#### 认证授权
- 手机号+密码注册/登录
- 游客模式（本地存储，限额体验）
- JWT Token认证，有效期7天
- 密码加密存储（bcrypt）
- 短信验证码防刷机制（60秒间隔，5次/天）
- 额度检查中间件（AI生成功能）
- 支持手机号与微信账号绑定/解绑
- JWT Token，有效期7天
- 支持Token刷新
- 敏感操作二次验证

#### 微信认证流程
```

│  客户端  │────▶│ 微信SDK │────▶│ 微信服务 │────▶│  服务端  │
└─────────┘     └─────────┘     └─────────┘     └─────────┘
     │               │               │               │
     │ 1.请求授权     │               │               │
     │──────────────▶│               │               │
     │               │ 2.拉起微信授权 │               │

     │               │ 3.返回授权码   │               │
     │               │◀──────────────│               │
     │ 4.发送授权码   │               │               │

**微信登录场景**:
| 场景 | 处理方式 |
|------|----------|
| 新用户首次微信登录 | 自动创建账号，生成随机昵称，引导绑定手机号 |
| 老用户微信登录 | 直接登录，同步用户数据 |
| 已绑定手机号的用户 | 可选择手机号或微信两种方式登录 |
| 解绑微信 | 需先绑定手机号，确保账号有至少一种登录方式 |
