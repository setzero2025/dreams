# 修复后端接口访问问题

## 问题描述
所有后端接口都访问不通，请求显示失败状态，大小为 0 B，响应时间长达 15-21 秒。

## 可能原因
1. 后端服务崩溃或未正常运行
2. 数据库连接问题导致请求阻塞
3. CORS 配置问题
4. 网络/防火墙问题
5. 代码编译/运行时错误

## 诊断步骤

### 步骤 1: 检查后端服务状态
- [ ] 查看后端终端输出，确认服务是否正常启动
- [ ] 检查是否有错误日志
- [ ] 测试健康检查接口 http://localhost:3002/health

### 步骤 2: 检查数据库连接
- [ ] 确认数据库连接是否稳定
- [ ] 检查连接池配置
- [ ] 查看是否有数据库连接阻塞请求

### 步骤 3: 检查 CORS 配置
- [ ] 确认 CORS 中间件正确配置
- [ ] 检查允许的域名列表

### 步骤 4: 检查代码错误
- [ ] 查看最近的代码修改是否引入错误
- [ ] 检查新添加的服务文件是否有语法错误
- [ ] 验证所有依赖是否正确导入

### 步骤 5: 测试基础连接
- [ ] 使用 curl 或浏览器直接访问健康检查接口
- [ ] 检查端口 3002 是否被占用
- [ ] 确认防火墙没有阻止连接

## 修复方案

### 方案 A: 如果是服务崩溃
1. 停止当前后端服务
2. 检查代码错误
3. 修复后重新启动

### 方案 B: 如果是数据库连接问题
1. 优化连接池配置
2. 添加连接重试机制
3. 确保数据库连接不会阻塞请求

### 方案 C: 如果是 CORS 问题
1. 检查并修复 CORS 配置
2. 确保前端域名在允许列表中

### 方案 D: 如果是代码错误
1. 检查最近修改的文件
2. 修复语法错误或运行时错误
3. 重新编译并启动

## 验证步骤
- [ ] 健康检查接口可访问
- [ ] 前端可以正常调用 API
- [ ] 响应时间在合理范围内（< 2秒）
